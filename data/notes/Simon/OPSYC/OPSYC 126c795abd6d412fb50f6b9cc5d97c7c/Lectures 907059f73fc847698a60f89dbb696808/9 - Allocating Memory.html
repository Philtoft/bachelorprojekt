1. Virtual Memory

Dynamic Memory Allocation

Allocator maintains a heap as a collection of variable-sized blocks, which are either allocated or free.

Type of allocators

  • Explicit allocator: application allocates and frees space. E.g. malloc and free in C.
  • Implicit allocator: application allocates, but does not free space. E.g. garbage collection in Java, ML, and Lisp.

In c, we deal with explicit memory allocation.

2. malloc

void foo(int n, int m) {
		int i, *p;

		/* Allocate a block of n ints */
		p = (int *) malloc(n * sizeof(int));
		if (p == NULL) {
				perror("malloc");
				exit(0);
		}

		/* Initialize allocated block */
		for (i=0; i<n; i++)
				p[i] = i;

		/* Return p to the heap */
		free(p);
}

Fragmentation

Poor memory utilization caused by fragmentation

  • internal fragmentation (fragmentation inside the memory block)
  • external fragmentation (fragmentation when no block is large enough to store a memory request)
